# Decision Assurance Framework

## Purpose

The Decision Assurance Framework (DAF) defines the five structural elements that every decision passing through the Decision Assurance Layer must contain. It is the conceptual contract of the system — the schema that unifies the data model, the UI, the AI prompt design, and the audit record.

Every `DecisionRecord` produced by this system is an instantiation of the framework.

---

## The Five Elements

### 1. Evidence Bundle

**What it is:** The raw, timestamped threat data that was available to both the AI and the analyst at the moment of decision.

**Why it matters:** Without a preserved evidence snapshot, post-incident review cannot determine whether a decision was reasonable given what was known at the time. Evidence that "drifts" — logs that are overwritten, alerts that expire — makes accountability impossible.

**Data fields (from `DecisionRecord` and scenario fixtures):**

| Field | Source | Description |
|---|---|---|
| `scenario_id` | Scenario fixture | Unique identifier linking the record to its originating alert |
| `scenario_title` | Scenario fixture | Human-readable name of the threat scenario |
| `raw_evidence` | Scenario fixture | List of timestamped log/telemetry strings (EDR, auth, proxy, network) |
| `alert_type` | Scenario fixture | Classification of the alert (e.g. Lateral Movement, Ransomware Precursor) |
| `mitre_tactic` / `mitre_technique` | Scenario fixture | MITRE ATT&CK mapping for the observed behaviour |
| `severity` | Scenario fixture | Assessed severity: LOW / MEDIUM / HIGH / CRITICAL |

**MVP implementation:** Evidence is captured in the scenario fixture JSON and displayed verbatim in the analyst workbench. It is referenced by `scenario_id` in the `DecisionRecord`.

---

### 2. AI Recommendation

**What it is:** The structured output of the AI analysis — not just a verdict, but the reasoning behind it and the actions it proposes.

**Why it matters:** A recommendation without reasoning is an oracle, not an assistant. Analysts need to understand *why* the AI reached its conclusion to apply independent judgment. Capturing the full recommendation — including reasoning and suggested actions — also provides material for model improvement and post-incident review.

**Data fields (from `DecisionRecord`):**

| Field | Type | Description |
|---|---|---|
| `ai_recommendation` | `str` | The recommended action: `"investigate"`, `"escalate"`, or `"dismiss"` |
| `ai_reasoning` | `str` | The AI's explanation of its recommendation in plain language |
| `ai_suggested_actions` | `List[str]` | 2-3 specific, concrete actions the AI proposes the analyst take |

**MVP implementation:** Generated by IBM watsonx.ai (Granite model) via a structured prompt. The model is instructed to return JSON with these exact fields.

---

### 3. Uncertainty Score

**What it is:** A numeric confidence score (0.0-1.0) that quantifies how certain the AI is about its recommendation.

**Why it matters:** AI systems frequently produce confidently-worded outputs regardless of their actual certainty. Making uncertainty explicit and visible — as a number and as a colour-coded signal — forces analysts to calibrate their trust in the recommendation. A 0.55 confidence score should trigger more scrutiny than a 0.92 score.

**Data fields (from `DecisionRecord`):**

| Field | Type | Description |
|---|---|---|
| `ai_confidence` | `float` | Score from 0.0 (no confidence) to 1.0 (full confidence) |

**UI interpretation:**

| Score Range | Badge Colour | Meaning |
|---|---|---|
| `>= 0.75` | Green | AI is reasonably confident — standard scrutiny appropriate |
| `0.50 - 0.74` | Amber | AI is uncertain — analyst should scrutinise evidence independently |
| `< 0.50` | Red | AI has low confidence — human judgment should dominate |

**MVP implementation:** Requested from Granite as part of the structured JSON response. The fallback mock response includes a realistic confidence value.

---

### 4. Human Judgment

**What it is:** The analyst's explicit, documented decision — not a passive approval click, but a structured record of what they decided and why.

**Why it matters:** Human judgment is the control that makes AI-assisted workflows legitimate. Without a formal judgment record, the "human in the loop" is a fiction. Capturing the rationale — especially for overrides — creates organisational learning and accountability.

**Data fields (from `DecisionRecord`):**

| Field | Type | Description |
|---|---|---|
| `analyst_id` | `str` | Identifier of the analyst (stubbed as `"analyst_01"` in MVP) |
| `analyst_action` | `str` | The analyst's decision: `"approve"`, `"reject"`, or `"override"` |
| `analyst_rationale` | `str` | Free-text explanation of the analyst's reasoning (required, not optional) |
| `override_description` | `Optional[str]` | If action is `"override"`: description of the alternative action taken |

**MVP implementation:** Collected via the Streamlit analyst action form. Rationale is a required field — the Submit button is disabled until it is filled. Override description is shown conditionally only when Override is selected.

**The three judgment options:**

- **Approve** — The analyst agrees with the AI recommendation and will act on it as stated.
- **Reject** — The analyst disagrees with the AI recommendation and will take no action (or a different undocumented action). The rationale explains why.
- **Override** — The analyst disagrees with the AI recommendation and will take a *specific different action*, which is documented in `override_description`.

---

### 5. Audit Record

**What it is:** The immutable, tamper-evident record that captures all four preceding elements as a single, versioned, hashed document persisted to IBM Cloud Object Storage.

**Why it matters:** The audit record is the DAL's primary deliverable. It is what makes AI-assisted decision-making reviewable, attributable, and defensible. The SHA-256 hash ensures that the record content cannot be silently modified after saving — any change to any field invalidates the hash.

**Data fields (from `DecisionRecord`):**

| Field | Type | Description |
|---|---|---|
| `record_id` | `str` | UUID4 unique identifier for this decision record |
| `timestamp` | `str` | UTC ISO-8601 timestamp of record creation |
| `record_hash` | `str` | SHA-256 hex digest computed over all other fields (64 hex chars) |

**Hash computation:** The hash is computed over the canonical JSON serialisation of all `DecisionRecord` fields *except* `record_hash` itself, with keys sorted for determinism. The hash is then added to the record before saving. Any post-save mutation of any field will produce a different hash, making tampering detectable.

**Storage:** Each record is persisted to IBM Cloud Object Storage under the key `decisions/{record_id}.json`. A local file fallback (`data/decisions/`) is provided for offline / demo use.

**MVP implementation:** Produced by `DecisionRecord.create()` in `src/decision_record.py`, saved by `src/cos_client.py`.

## Framework Summary

```
+------------------------------------------------------------------+
|                   DECISION ASSURANCE LAYER                       |
|                                                                  |
|  [1. Evidence Bundle]  ->  [2. AI Recommendation]               |
|         |                         |                             |
|                  [3. Uncertainty Score]                          |
|                         |                                        |
|              [4. Human Judgment]                                 |
|         (approve / reject / override + rationale)               |
|                         |                                        |
|              [5. Audit Record]                                   |
|         (SHA-256 hashed, persisted to IBM COS)                   |
+------------------------------------------------------------------+
```

|---|---|---|---|---|
| Evidence Bundle | `scenario_id`, `scenario_title` | `id`, `title`, `raw_evidence`, `severity`, `mitre_*` | Evidence panel (left column) | This document, S1 |
| AI Recommendation | `ai_recommendation`, `ai_reasoning`, `ai_suggested_actions` | — | Recommendation card (right column) | This document, S2 |
| Uncertainty Score | `ai_confidence` | — | Confidence badge (colour-coded) | This document, S3 |
| Human Judgment | `analyst_action`, `analyst_rationale`, `override_description`, `analyst_id` | — | Analyst action form | This document, S4 |
| Audit Record | `record_id`, `timestamp`, `record_hash` | — | Decision confirmation banner + Audit Log tab | This document, S5 |
